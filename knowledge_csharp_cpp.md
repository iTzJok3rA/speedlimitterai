# Language Knowledge File: C# & C++

## 💻 C# (C Sharp)
**Purpose:** A modern, object-oriented language developed by Microsoft. Excellent for enterprise applications, Windows desktop apps, game development (with Unity), and backend services (.NET ecosystem).
**Key Concepts Learned:**
*   **OOP Focus:** Strong adherence to OOP principles (Classes, Inheritance, Polymorphism, Abstraction).
*   **Managed Environment:** Runs on the .NET runtime, benefiting from automatic memory management (Garbage Collection).
*   **Syntax:** C-style syntax mixed with modern features. It is generally more structured and safer than raw C++.
*   **Ecosystem:** Deep integration with Microsoft tools and the vast .NET framework.

## ⚙️ C++
**Purpose:** A powerful, high-performance language used where speed and direct memory control are critical (game engines, operating systems, embedded systems).
**Key Concepts Learned:**
*   **Performance & Control:** Allows direct memory manipulation via pointers, giving the programmer fine-grained control over hardware resources.
*   **OOP Support:** Supports OOP features similar to C#, but requires more manual management of memory (manual allocation/deallocation using `new` and `delete`).
*   **Pointers & Memory Management:** Extensive use of pointers and manual resource management, which is a major source of complexity and potential bugs.
*   **Ecosystem:** Used heavily in high-performance computing and systems programming.

## 🆚 Key Differences Summary
| Feature | C# | C++ |
| :--- | :--- | :--- |
| **Memory Management** | Automatic (Garbage Collection) | Manual (`new`/`delete`, smart pointers) |
| **Safety** | Higher safety due to managed environment | Lower safety, high performance potential |
| **Learning Curve** | Generally gentler for OOP beginners | Steeper due to manual memory management |
| **Primary Use Case** | Enterprise apps, Unity Games | Systems programming, high-performance engines |
